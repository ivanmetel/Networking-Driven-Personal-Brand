# Разбор YAML-конфигурации

Этот блок YAML (Frontmatter) принадлежит файлу описания **методологической гипотезы** (Hypothesis) или **архитектурного решения** в рамках семейства **F5** (Архитектура продукта/Целевой системы). В текущей файловой структуре такие файлы обычно располагаются в `content/2.System-of-Interest/2.2.Architecture/` или специальной директории для гипотез.

Мной был произведен поиск данного файла (`HYP-034`) в проекте, но он **не был найден**. Вероятно, это новый документ или черновик.

## Детальный разбор строк

```yaml
id: HYP-034
```
**Назначение:** Уникальный идентификатор документа.
- `HYP` указывает на тип "Гипотеза" (Hypothesis).
- `034` — порядковый номер. Используется для жестких ссылок из других документов.

```yaml
title: Перенос за 7–14 дней повышает устойчивость ритма
```
**Назначение:** Человекочитаемый заголовок документа. Отражает суть гипотезы или изменения.

```yaml
type: doc
```
**Назначение:** Технический тип объекта в системе.
- `doc` — стандартный документ (в отличие от `index`, `person`, `event` и т.д.).

```yaml
family: F5
```
**Назначение:** Принадлежность к семейству документов по модели SRT (System-Role-Table).
- **F5** = **Target System (System-of-Interest) × Engineer (Architecture)**.
- Это означает, что документ описывает **устройство/архитектуру** целевой системы (продукта) и предназначен для архитекторов и методологов.

```yaml
developer: Методсовет
```
**Назначение:** Указание роли или группы, разработавшей документ (в отличие от конкретного `owner`).

```yaml
target_audience:
  - Наставники
  - Методологи
```
**Назначение:** Для кого предназначен этот документ (кто должен его читать).

```yaml
audience: human
```
**Назначение:** Указание, что текст оптимизирован для чтения человеком (а не AI-агентом).

```yaml
edit_mode: manual
```
**Назначение:** Режим редактирования. `manual` означает, что документ правится вручную людьми, и его нельзя перезаписывать автогенераторами без спроса.

```yaml
layer: methodology
```
**Назначение:** Слой архитектуры. Указывает, что документ относится к методологическому уровню (правила, логика), а не к коду или инфраструктуре.

```yaml
scope: global-core
```
**Назначение:** Область действия. `global-core` — изменение затрагивает ядро системы глобально.

```yaml
security: public
```
**Назначение:** Уровень доступности (публичный).

```yaml
ai_role: none
```
**Назначение:** Роль ИИ в создании/использовании документа. `none` — ИИ не задействован.

```yaml
links:
  problem: PRB-021
  intervention: INT-034a
  requirements:
    - REQ-127
  architecture:
    - ARC-STAT-07
  events:
    - DAT-EVT-17: unit.applied_7_14d
  metrics:
    - IND.2.1.1
```
**Назначение:** Семантические связи с другими объектами системы (traceability).
- `problem: PRB-021` — какую проблему решает.
- `intervention: INT-034a` — какое вмешательство описывает.
- `requirements` — какие требования реализует.
- `architecture` — связь с архитектурными элементами (статистика).
- `events` — привязка к событийной модели данных (аналитика unit.applied...).
- `metrics` — на какой показатель (индикатор) влияет.

```yaml
depends_on:
  - Концепция созидателя 2.2
```
**Назначение:** Зависимости. Документ или концепция, без которой эта гипотеза не имеет смысла или не может быть реализована.

```yaml
affects:
  - Программы развития 2.2
```
**Назначение:** Влияние. Что изменится или что нужно обновить при принятии этой гипотезы.

```yaml
version: 0.2
```
**Назначение:** Версия документа.

```yaml
owner: Методсовет
```
**Назначение:** Ответственный владелец документа (кто принимает решения по нему).

```yaml
review:
  - Этика
  - Онтолог
```
**Назначение:** Кто должен провести ревью документа перед утверждением.

```yaml
status: draft
```
**Назначение:** Текущий жизненный цикл документа (черновик).

```yaml
created: 2026-01-07, 11:22:18
updated: 2026-01-07, 11:22:18
```
**Назначение:** Временные метки создания и последнего обновления.
