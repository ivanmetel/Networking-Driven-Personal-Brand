# Ядро B: Product — Наша система

## Назначение

Документация о создании **Приложения** для поддержки ORA.

## Системы ядра

| Код | Система | Вопрос | Описание |
|-----|---------|--------|----------|
| **B1** | Организатор ORA | Для кого? | Надсистема для нашей системы: организатор + площадка + инструменты |
| **B2** | Приложение | Что создаём? | Наша система: платформа для ORA |
| **B3** | Компания Planora | Кто создаёт? | Система создания нашей системы: ИТ-служба + ORA-студия |

## Структура

```
B.Product/
├── B1.Suprasystem/              # Организатор ORA
│   ├── B1.1.Предприниматель/    # Ценность для организатора
│   ├── B1.2.Инженер/            # Структура организатора ORA
│   └── B1.3.Менеджер/           # Workflow организатора
│
├── B2.System-of-Interest/       # Приложение
│   ├── B2.1.Предприниматель/    # Требования
│   ├── B2.2.Инженер/            # Архитектура
│   └── B2.3.Менеджер/           # Использование
│
└── B3.Constructor/              # Компания Planora
    ├── B3.1.Предприниматель/    # Миссия
    ├── B3.2.Инженер/            # ИТ + ORA-студия
    └── B3.3.Менеджер/           # Процессы
```

## Связь с ядром A

Приложение (B2) используется в ORA (A3) как инструмент цифрового контура.

> См. [F0. Repository-management](../../Repository-management/) — раздел "Мосты между ядрами" в [document-families.md](../../Repository-management/0.1.%20Логика%20хранилища%20и%20знаний/document-families.md).

## Для кого это ядро

- **ИТ-служба** — разработчики, DevOps, архитекторы
- **Product Owner** — управление требованиями
- **ORA-студия** — внедрение и обучение
