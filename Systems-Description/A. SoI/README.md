# Ядро A: Target — Целевая система

## Назначение

Документация о создании **впечатлений у участников мероприятий**.

## Системы ядра

| Код | Система | Вопрос | Описание |
|-----|---------|--------|----------|
| **A1** | Среда человека | В чём живёт? | Надсистема: семья, сообщество, город, инфосреда |
| **A2** | Впечатлённый человек | Что создаём? | Целевая система: человек с повышенной впечатлённостью |
| **A3** | ORA | Кто создаёт? | Система создания: мероприятие-как-продукт |

## Структура

```
A.Target/
├── A1.Suprasystem/              # Среда человека
│   ├── A1.1.Предприниматель/    # Зачем человеку мероприятия
│   ├── A1.2.Инженер/            # Структура среды
│   └── A1.3.Менеджер/           # Поведение человека
│
├── A2.System-of-Interest/       # Впечатлённый человек
│   ├── A2.1.Предприниматель/    # Что такое впечатлённость
│   ├── A2.2.Инженер/            # Компоненты впечатления
│   └── A2.3.Менеджер/           # Путь участника
│
└── A3.Constructor/              # ORA (ивент-продукт)
    ├── A3.1.Предприниматель/    # Философия ORA
    ├── A3.2.Инженер/            # Структура и описания ORA
    └── A3.3.Менеджер/           # Процесс создания ORA
```

## Связь с ядром B

ORA (A3) использует **Приложение** (B2) как инструмент цифрового контура.

> См. [F19. Repository-management](../../Repository-management/) — раздел "Мосты между ядрами" в [document-families.md](../../Repository-management/0.1.%20Логика%20хранилища%20и%20знаний/document-families.md).

## Для кого это ядро

- **ORA-студия** — методологи, проектировщики мероприятий
- **Организаторы** — те, кто создаёт и проводит ORA
- **Фасилитаторы** — те, кто работает с участниками на мероприятии
