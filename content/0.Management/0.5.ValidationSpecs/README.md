# ТЗ для автоматической проверки документов (0.5.ValidationSpecs)

## Назначение раздела

Здесь размещаются **технические задания** для автоматической проверки документов хранилища. Каждое ТЗ описывает правила валидации, которые должен выполнять AI-агент.

## Типы проверок

### 1. Структурные проверки

| Проверка | Описание | Критичность |
|----------|----------|-------------|
| Frontmatter | Наличие и корректность YAML | Критическая |
| Путь файла | Соответствие system/role | Критическая |
| Имя файла | Соблюдение naming convention | Высокая |
| Структура | Наличие обязательных разделов | Высокая |

### 2. Связностные проверки

| Проверка | Описание | Критичность |
|----------|----------|-------------|
| Related-ссылки | Существование связанных файлов | Высокая |
| Двунаправленность | Обратные ссылки в related | Средняя |
| Сироты | Документы без входящих ссылок | Низкая |

### 3. Контентные проверки

| Проверка | Описание | Критичность |
|----------|----------|-------------|
| FPF-соответствие | Применение заявленных паттернов | Средняя |
| Терминология | Соответствие глоссарию | Низкая |
| Дублирование | Повторение контента | Низкая |

## Документы раздела

- [frontmatter-spec.md](frontmatter-spec.md) — ТЗ проверки frontmatter
- [links-spec.md](links-spec.md) — ТЗ проверки ссылок
