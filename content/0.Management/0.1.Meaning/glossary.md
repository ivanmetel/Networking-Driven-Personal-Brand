# Глоссарий хранилища

## Назначение документа

Определения ключевых терминов, используемых в хранилище. Термины определены в контексте данного репозитория (Bounded Context).

## Основные термины

### SRT-метод

#### SRT (Systems-Roles-Table)
Метод организации знаний через таблицу 3×3: 3 системы × 3 роли + метасемейство F0.

#### Система (System)
Один из четырёх уровней организации: Management, Supersystem, TargetSystem, CreationSystem.

#### Роль (Role)
Одно из трёх измерений: Meaning (смысл), Architecture (архитектура), Operations (операции).

#### Ячейка матрицы
Пересечение системы и роли. Пример: `2.TargetSystem/2.2.Architecture`.

### FPF-термины

#### FPF (First Principles Framework)
"Операционная система для мышления" — архитектура трансдисциплинарного рассуждения.

#### Холон (Holon)
Сущность, которая одновременно является целым и частью большей системы (паттерн A.1).

#### Ограниченный контекст (Bounded Context)
Область, в которой термин имеет определённое значение (паттерн A.1.1).

#### Мост (Bridge)
Явная связь между терминами в разных контекстах с указанием потерь перевода (паттерн F.9).

#### F-G-R (Formality-Scope-Reliability)
Кортеж доверия: формальность (0-9), область применения, надёжность (0.0-1.0).

#### Эпистема (Episteme)
Артефакт знания: документ, модель, описание. Противопоставляется системе (физическому объекту).

### Документные термины

#### Frontmatter
YAML-блок в начале Markdown-документа с метаданными.

#### Статус документа
Уровень зрелости: stub → draft → active → archived.

#### SSOT (Single Source of Truth)
Принцип единственного источника истины — каждый факт определён в одном месте.

### Организационные термины

#### Контрибьютор
Участник, вносящий вклад в репозиторий.

#### AI-агент
Автоматизированный помощник (Claude Code), работающий с репозиторием.

#### Workflow
Последовательность шагов для выполнения задачи.

## Термины по системам

### Management (Управление)
| Термин | Определение |
|--------|-------------|
| Метасистема | Система, управляющая другими системами |
| Процесс | Повторяемая последовательность действий |

### Supersystem (Надсистема)
| Термин | Определение |
|--------|-------------|
| Контекст | Внешняя среда, влияющая на проект |
| Проектная роль | Заинтересованная сторона, играющая определённую роль в проекте |

### TargetSystem (Целевая система)
| Термин | Определение |
|--------|-------------|
| Продукт | Создаваемая система/артефакт |
| Требование | Ожидаемое свойство продукта |

### CreationSystem (Система создания)
| Термин | Определение |
|--------|-------------|
| Инструмент | Средство для создания продукта |
| Практика | Способ выполнения работы |

## Добавление терминов

### Критерии добавления

Термин добавляется, если:
1. Используется в нескольких документах
2. Имеет специфическое значение в контексте хранилища
3. Отличается от общеупотребительного значения

### Шаблон термина

```markdown
#### Название термина
Определение в 1-2 предложениях.

**Контекст:** Где применяется
**FPF-связь:** Паттерн (если есть)
**См. также:** Связанные термины
```
