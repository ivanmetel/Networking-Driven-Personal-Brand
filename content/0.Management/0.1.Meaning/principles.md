# Принципы организации хранилища

## Назначение документа

Этот документ определяет фундаментальные принципы организации хранилища знаний. Принципы применяются ко всем разделам и документам.

## Основные принципы

### 1. Единственное место для каждого документа (SSOT)

**Принцип:** Каждый документ имеет одно каноническое место в структуре.

**Обоснование:** Дублирование ведёт к рассинхронизации и конфликтам версий.

**Применение:**
- Используй трёхуровневую систему: Система → Роль → Документ
- Ссылайся на документы, не копируй содержимое
- При необходимости кросс-ссылок используй `related` в frontmatter

**FPF-связь:** Паттерн A.1.1 (Bounded Context) — каждый контекст имеет единственное определение термина.

### 2. Разделение типов информации

**Принцип:** Документы классифицируются по типу и назначению.

**Типы документов:**

| Тип | Описание | Пример |
|-----|----------|--------|
| `doc` | Концептуальный документ | Принципы, описания |
| `spec` | Спецификация | API-схема, модель данных |
| `process` | Процессный документ | Workflow, инструкция |
| `report` | Отчёт | Аналитика, статус |
| `template` | Шаблон | Образец документа |

**Применение:**
- Указывай `type` в frontmatter
- Размещай в соответствующей роли (Meaning/Architecture/Operations)

### 3. Роль человека и AI

**Принцип:** Чёткое разделение ответственности между человеком и AI.

| Актор | Ответственность |
|-------|-----------------|
| **Человек** | Стратегия, архитектура, принятие решений, валидация |
| **AI** | Производные артефакты, рутинные задачи, поиск, анализ |

**Применение:**
- Человек редактирует ключевые документы (README, принципы, архитектура)
- AI помогает с генерацией, рефакторингом, проверкой связей
- Критические изменения требуют человеческого review

**FPF-связь:** Паттерн A.12 (External Transformer) — внешний агент изменяет систему.

### 4. Строительство на существующем знании

**Принцип:** Новые документы создаются на основе существующих шаблонов и терминов.

**Применение:**
- Используй шаблоны из `README.md` каждой папки
- Ссылайся на глоссарий для терминов
- Проверяй существующие документы перед созданием нового

### 5. Богатые метаданные

**Принцип:** Каждый документ имеет структурированные метаданные в YAML frontmatter.

**Обязательные поля:**
- `type` — тип документа
- `status` — статус (stub, draft, active, archived)
- `created`, `updated` — даты
- `system`, `role` — место в SRT-таблице

**Рекомендуемые поля:**
- `target_audience` — целевая аудитория
- `related` — связанные документы
- `fpf_principles` — применяемые принципы
- `fpf_patterns` — паттерны FPF

**FPF-связь:** Паттерн B.3 (Trust Calculus) — метаданные формируют F-G-R оценку.

### 6. Уровни зрелости документов

**Принцип:** Документы проходят через уровни зрелости.

| Статус | Описание | Можно использовать |
|--------|----------|-------------------|
| `stub` | Заглушка, идея | Нет |
| `draft` | Черновик | С осторожностью |
| `active` | Актуальный | Да |
| `archived` | Архив | Для истории |

**Применение:**
- Новые документы начинаются как `draft`
- Переход в `active` после review
- Устаревшие документы → `archived`, не удаляются

### 7. Контекстуальность

**Принцип:** Значение термина определяется контекстом (папкой).

**Применение:**
- Термин может иметь разное значение в разных системах
- Явно указывай контекст при использовании термина вне его папки
- Используй мосты (bridges) для связи терминов между контекстами

**FPF-связь:** Паттерн A.1.1 (Bounded Context) + F.9 (Bridges).

### 8. Связность документов

**Принцип:** Документы образуют связную сеть через явные ссылки.

**Применение:**
- Указывай `related` в frontmatter
- Поддерживай двунаправленные связи
- Проверяй "сироты" (документы без связей)

### 9. Простота формата

**Принцип:** Markdown + YAML frontmatter как единый формат.

**Обоснование:**
- Читаемость человеком и машиной
- Версионирование в Git
- Совместимость с любыми инструментами

### 10. Этика и ответственность

**Принцип:** Документы не должны содержать вредоносного или неэтичного контента.

**Применение:**
- Не храни секреты в репозитории
- Указывай авторство и источники
- Соблюдай лицензии
