# Архив (0.99.Archive)

## Назначение раздела

Хранилище **устаревших документов** и материалов, которые больше не актуальны, но могут понадобиться для истории.

## Правила архивации

### Когда архивировать

- Документ устарел и заменён новым
- Функционал/проект закрыт
- Информация перенесена в другое место
- Временный документ выполнил свою задачу

### Процесс архивации

1. Изменить `status: archived` в исходном файле
2. Переместить файл в `0.99.Archive/`
3. Добавить запись о причине архивации
4. Обновить ссылки в связанных документах

### Структура архива

```
0.99.Archive/
├── README.md
├── 2026-01/                    # По месяцам
│   ├── old-requirements.md
│   └── deprecated-api.md
└── 2025-12/
    └── initial-plan.md
```

## Формат архивного документа

```markdown
---
type: doc
status: archived
created: YYYY-MM-DD
archived: YYYY-MM-DD
archived_reason: "Заменён документом X"
original_path: "content/2.TargetSystem/..."
system: "Management"
role: "Archive"
---

# [Оригинальное название] [ARCHIVED]

> **Архивировано:** YYYY-MM-DD
> **Причина:** [Причина архивации]
> **Заменён:** [Ссылка на новый документ, если есть]

---

[Оригинальное содержимое]
```

## Правила работы с архивом

### Можно

✅ Читать для понимания истории
✅ Извлекать полезную информацию
✅ Ссылаться из новых документов

### Нельзя

❌ Редактировать архивные документы
❌ Использовать как актуальный источник
❌ Удалять без особой необходимости

## Периодическая очистка

Раз в квартал:
1. Просмотреть архив старше 1 года
2. Оценить ценность для истории
3. Удалить явно ненужное
4. Объединить похожие документы
