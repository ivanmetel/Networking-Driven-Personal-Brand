# Раздел 0. Управление (Management)

## Назначение раздела

Раздел **Management** — это **метасистема хранилища** (панель управления), которая управляет логикой организации знаний, рабочими процессами и автоматизацией.

В терминах FPF (паттерн A.1 — Holonic Foundation): Management — холон, который одновременно является частью большей системы (проекта) и содержит свои подсистемы.

Management выполняет **восемь ключевых функций**:

## Детали подразделов

### 0.1. Meaning — Логика и принципы

**Вопрос:** Зачем? Что значит?

Фундаментальные основы организации хранилища: принципы, терминология, классификация. "Конституция" репозитория.

**Ключевые документы:**
- `principles.md` — 10+ принципов организации
- `glossary.md` — определения терминов SRT и FPF
- `taxonomy.md` — система классификации и тегов

### 0.2. Architecture — Структура

**Вопрос:** Как устроено?

Архитектура хранилища: структура папок, роли участников, стандарты оформления.

**Ключевые документы:**
- `structure.md` — полная структура content/
- `roles.md` — RACI-матрица ответственности
- `standards.md` — форматирование и frontmatter

### 0.3. Operations — Процессы работы

**Вопрос:** Как работает?

Операционные процессы: workflows, создание документов, работа с AI.

**Ключевые документы:**
- `workflows.md` — процессы создания/редактирования
- `claude-fpf.md` — интеграция Claude Code с FPF
- `document-creation.md` — пошаговая инструкция

### 0.4. AutoReports — Автоматические отчёты AI

Автоматически генерируемые отчёты о состоянии хранилища:
- Качество документов
- Покрытие таблицы SRT
- Битые ссылки
- FPF-соответствие

### 0.5. ValidationSpecs — ТЗ для автопроверок

Технические задания для автоматической валидации документов:
- Проверка frontmatter
- Проверка ссылок
- Проверка соответствия FPF

### 0.6. AccumulationGuide — Процесс накопления

Руководство по постепенному развитию хранилища:
- Этапы накопления (0-3)
- Метрики роста
- Еженедельный цикл

### 0.9. Incoming — Входящие

Inbox для новых идей, черновиков и материалов, которые ещё не нашли место в структуре.

### 0.99. Archive — Архив

Хранилище устаревших документов для сохранения истории.

## Принципы работы с разделом

1. **SSOT** — каждая сущность имеет одно место определения
2. **Метаданные обязательны** — YAML frontmatter для всех документов
3. **Разделение ролей** — люди: стратегия, AI: рутина
4. **Инкрементальное развитие** — не создавай документы "на будущее"

## FPF-интеграция

| Паттерн | Применение |
|---------|------------|
| **A.1 Holonic Foundation** | Каждый документ — холон в иерархии |
| **A.1.1 Bounded Context** | Каждая папка — ограниченный контекст |
| **A.7 Strict Distinction** | Чёткое различение: документ ≠ процесс |
| **B.3 Trust Calculus** | F-G-R метрики для оценки документов |
| **B.4 Canonical Evolution** | Процесс накопления хранилища |
