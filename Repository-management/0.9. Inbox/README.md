# 0.9. Inbox

## Назначение раздела

**Inbox** для новых идей, черновиков и материалов, которые ещё не нашли своё место в структуре хранилища.

**Ключевой вопрос:** Что нужно обработать?

## Правила использования

### Что сюда помещать

✅ Сырые заметки и идеи
✅ Черновики документов
✅ Внешние материалы для обработки
✅ Вопросы для исследования
✅ Предложения по улучшению

### Что НЕ помещать

❌ Готовые документы (→ в соответствующую ячейку)
❌ Устаревшие материалы (→ в Архив)
❌ Секреты и приватные данные

## Формат входящих

```markdown
---
type: doc
status: stub
created: YYYY-MM-DD
system: "Management"
target_system: "?"        # Предполагаемая система
target_family: "?"        # Предполагаемое семейство F1-F9
---

# [Рабочее название]

## Идея

[Краткое описание идеи или материала]

## Источник

[Откуда взято / кем предложено]

## Следующие шаги

- [ ] Определить место в структуре
- [ ] Оформить по стандартам
- [ ] Переместить в целевую папку
```

## Workflow обработки входящих

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  Входящее   │───▶│  Triage     │───▶│  Место      │
│  поступило  │    │  (разбор)   │    │  определено │
└─────────────┘    └─────────────┘    └──────┬──────┘
                                             │
                   ┌─────────────────────────┼─────────────────────────┐
                   ▼                         ▼                         ▼
           ┌─────────────┐           ┌─────────────┐           ┌─────────────┐
           │  Оформить   │           │  Отклонить  │           │  Архивировать│
           │  и перенести│           │  (удалить)  │           │  (0.99)      │
           └─────────────┘           └─────────────┘           └─────────────┘
```

## Еженедельный review

Каждую неделю:
1. Просмотреть все входящие
2. Для каждого определить действие:
   - **Оформить** → перенести в целевую папку
   - **Отложить** → оставить в Входящие
   - **Отклонить** → удалить
   - **Архивировать** → перенести в Архив
3. Цель: не более 10 элементов в Входящие
