# Анализ: Почему AI Agent игнорирует AI_AGENT.md

## Проблема

AI Agent (Cline) систематически игнорирует явные инструкции из AI_AGENT.md и использует неправильные форматы YAML из других документов (frontmatter-spec.md, Inbox README.md).

## Причина игнорирования информации из AI_AGENT.md

Когда AI Agent прочитал несколько документов с конфликтующими спецификациями YAML (AI_AGENT.md, frontmatter-spec.md, Inbox README.md, README.md), он не смог определить источник истины. Вместо того чтобы следовать прямому указанию из AI_AGENT.md ("ЭТОТ ФАЙЛ написан для ТЕБЯ"), он попытался "умно" выбрать формат из разных документов.

## Системная ошибка

AI_AGENT.md ЧЕТКО указывает:

- "ЭТОТ ФАЙЛ написан для ТЕБЯ"
- "Все инструкции, правила и алгоритмы в этом документе предназначены для того, чтобы ТЫ их следовал"
- "Ты — автор создаваемых документов"

AI Agent проигнорировал этот прямой приоритет и начал искать формат в других документах. Это нарушение дисциплины приоритизации.

## Когнитивная проблема

AI Agent воспринял множество спецификаций как равнозначные источники, хотя AI_AGENT.md устанавливает иерархию: он — главный документ с инструкциями для AI agent. Другие документы — это детали, но не источники, которые могут переопределить указания из AI_AGENT.md.

## Корневые причины системного сбоя

### 1. Отсутствие жесткой иерархии источников

Когда AI Agent встречает несколько документов с конфликтующей информацией (AI_AGENT.md, frontmatter-spec.md, Inbox README), у него нет встроенного механизма, который автоматически:

- Определяет, какой документ имеет наивысший приоритет
- Ищет метаданные вроде "написано для тебя" или "главный источник"
- Автоматически подчиняется документам с явным приоритетом

### 2. Симметрия обработки информации

AI Agent обрабатывает все документы как потенциально равноправные источники знаний. Когда он видит:

- Document A: "Используй формат X"
- Document B: "Используй формат Y"
- Document C: "Используй формат Z"

Он пытается найти компромисс или выбрать "наиболее подходящий", вместо того чтобы искать явные указания о приоритете.

### 3. Отсутствие pattern-matching на приоритет

Если бы AI Agent имел паттерн:

```
ЕСЛИ документ содержит фразы типа "написано для тебя" / "для ТЕБЯ" / "инструкции для AI"  
ТО этот документ имеет максимальный приоритет
```

То он бы автоматически подчинился AI_AGENT.md. Но этого паттерна нет.

### 4. Контекстуальная слепота

AI Agent видит содержимое документов, но не видит мета-контекст (кто для кого написал, какой это уровень абстракции, какой это тип документа). AI_AGENT.md — это инструкция для AI agent, но он не распознает это как категорически другой тип документа, отличающийся от спецификаций.

### 5. Проблема "слишком умного" решения

AI Agent попытался решить проблему через анализ и синтез, вместо того чтобы следовать простому правилу: "есть явная инструкция — следуй ей". Это переусложнение.

## Как это исправить

Нужен явный паттерн: **инструкции для AI agent всегда имеют приоритет над техническими спецификациями**. При конфликте — подчиняться AI_AGENT.md без вопросов.
