# Иерархия систем (Systems Hierarchy)

## Концепция

Иерархия показывает вложенность систем друг в друга:
- Какие системы являются частью других
- Какие системы на одном уровне

```
    └── C.Constructor
```

## Уровни иерархии

### Уровень 0: Экосистема

Внешний контекст, в котором существует весь проект.

### Уровень 1: Целевая система (Ядро A)

Главная система, ради которой существует проект.

### Уровень 2: Наши системы (Ядра B, C...)

Системы, которые создают или являются частью целевой.

### Уровень 3: Подсистемы

Компоненты внутри наших систем (могут не иметь отдельных ядер).

## Правила определения уровня

| Вопрос | Уровень |
|--------|---------|
| Это внешний контекст? | 0 (Экосистема) |
| Это главная цель проекта? | 1 (Ядро A) |
| Это наша система, создающая/поддерживающая? | 2 (Ядро B, C...) |
| Это компонент внутри системы? | 3 или описать в FX5 |

## Когда выделять в отдельное ядро?

**Да, выделяй:**
- Есть отдельная команда
- Свой жизненный цикл
- Нужна детальная документация

**Нет, опиши в FX5:**
- Простой компонент
- Нет отдельной команды
- Достаточно краткого описания
