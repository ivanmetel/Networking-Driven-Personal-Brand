# Роли и ответственность

## Назначение документа

Определение ролей участников работы с хранилищем и их ответственности.

## Основные роли

### 1. Владелец репозитория (Owner)

**Ответственность:**
- Стратегическое направление развития
- Утверждение архитектурных решений
- Управление доступом
- Финальное решение по спорным вопросам

**Права:**
- Полный доступ ко всем разделам
- Мержить без review
- Изменять настройки репозитория

### 2. Администратор (Administrator)

**Ответственность:**
- Поддержка структуры хранилища
- Review изменений в Management разделе
- Управление CI/CD
- Документирование процессов

**Права:**
- Редактирование всех разделов
- Управление ветками
- Настройка автоматизации

### 3. Контрибьютор (Contributor)

**Ответственность:**
- Создание и редактирование документов
- Соблюдение стандартов оформления
- Поддержка связей между документами
- Участие в review

**Права:**
- Создание веток
- Создание PR
- Редактирование документов (через PR)

### 4. AI-агент (AI agent Code)

**Ответственность:**
- Помощь в создании документов
- Проверка соответствия FPF
- Анализ структуры и связей
- Рефакторинг по запросу

**Права:**
- Чтение всех документов
- Создание PR (через человека)
- Анализ и рекомендации

**Ограничения:**
- Не принимает стратегических решений
- Критические изменения требуют human review
- Не имеет доступа к секретам

## Матрица ответственности (RACI)

| Действие | Owner | Admin | Contributor | AI |
|----------|-------|-------|-------------|-----|
| Изменение структуры | A | R | C | I |
| Создание документа | I | C | R | C |
| Review PR | A | R | R | C |
| Обновление FPF | A | R | C | C |
| Рефакторинг | A | R | R | R |
| Удаление документа | A | R | C | I |

**Легенда:**
- **R** — Responsible (выполняет)
- **A** — Accountable (ответственный)
- **C** — Consulted (консультируется)
- **I** — Informed (информируется)

## Процесс назначения ролей

### Присвоение роли

1. Запрос на роль через Issue
2. Review владельцем
3. Добавление в CODEOWNERS (если применимо)
4. Обновление документации

### Отзыв роли

1. Решение владельца
2. Удаление из CODEOWNERS
3. Информирование участника

## Распределение ответственности по ядрам

Ответственность партнёров за системные ядра хранилища:

| Партнёр | Зона ответственности | Ядра/Разделы |
|---------|---------------------|--------------|
| **Лари Церенов** | Целевая система + Надсистема продукта | A (полностью) + B1 |
| **Андрей Иванов** | Продукт и его создание | B2 + B3 |

### Детализация

**Лари Церенов:**
- `A.Target/` — полное ядро A (Впечатлённый человек, Среда, ORA)
- `B.Product/B1.Suprasystem-ORA-Organization/` — Организатор ORA как надсистема

**Андрей Иванов:**
- `B.Product/B2.Target-system-App/` — Приложение (наша система B)
- `B.Product/B3.Constructor-Planora/` — Компания Planora (система создания нашей системы B)
